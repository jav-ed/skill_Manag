Format notes:
- The opening summary is required. A linker without it is a pure index, which forces the reader to open every link to know what is relevant. The summary lets the reader skip the area entirely when it does not apply.
- Separate the link from its description with a colon, never an em-dash.
- The description is as long as it needs to be. One line is fine when one line covers it. Five lines is fine when the topic genuinely needs five lines. The goal is clarity for navigation, not brevity. A reader should be able to decide whether to click without opening the file. If the topic has multiple distinct angles, list them all rather than collapsing to a vague summary.
- If the folder has sub-folders with their own linkers, link to those sub-linkers, not the leaf files inside them.
